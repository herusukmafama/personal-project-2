# JSON Schema - Help & Support Output

## Purpose

This document describes the JSON structure generated by the browser converter for Help & Support DOCX templates.

## Root Object

```json
{
  "page_title": "Job Data Syncronization",
  "breadcrumb": [],
  "page_content": {}
}
```

## Fields

### `page_title`

- Type: `string`
- Source: `Features` value from the metadata table.
- Fallback: first `Heading 1`, then uploaded filename without extension.

### `breadcrumb`

- Type: `array`
- Items:

```json
{
  "label": "Help and Support",
  "url": "helpAndSupport"
}
```

Expected order:

1. Help and Support root.
2. Feature category.
3. Current page title.

### `page_content`

- Type: `object`
- Required fields:
  - `type`
  - `navigations`
  - `section`

Example:

```json
{
  "type": "content",
  "navigations": [
    {
      "label": "Overview",
      "id": "overview"
    }
  ],
  "section": []
}
```

## Section Object

```json
{
  "title": "Job Data Syncronization",
  "id": "overview",
  "description": "<p>...</p>",
  "data": []
}
```

## Content Item Types

### Basic Content

```json
{
  "title": "Feature",
  "id": "",
  "description": "<p>...</p>",
  "data": []
}
```

### Document Item

```json
{
  "title": "",
  "description": "<p>...</p>",
  "order": 1,
  "links": [],
  "type": "document"
}
```

### Image Item

```json
{
  "title": "",
  "description": "",
  "order": 1,
  "image": [
    {
      "url": "core/050073-figure-01.png",
      "notes": ""
    }
  ],
  "type": "image"
}
```

### Tips Item

```json
{
  "title": "Tips",
  "description": "<p>...</p>",
  "order": 3,
  "type": "tips"
}
```

### Forbidden Item

```json
{
  "title": "Limitation",
  "description": "<p><ul>...</ul></p>",
  "order": 2,
  "type": "forbidden"
}
```

### Full Documentation Item

```json
{
  "title": "Full Documentation",
  "id": "",
  "description": "Untuk informasi detail, silakan akses dokumen berikut",
  "data": [
    {
      "url": "core/050073-Job-Data-Syncronization.pdf",
      "label": "Job Data Syncronization"
    }
  ]
}
```

## Current Constraints

- The converter currently targets the Help & Support DOCX template represented by `samples/word/sample_01.docx`.
- HTML descriptions are intentionally preserved because downstream Help & Support content expects formatted strings.
- Image and PDF files are referenced by generated URL only. The browser app does not store or extract files.
